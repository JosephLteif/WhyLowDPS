# Security Policy

## Supported versions

Security updates are provided for the latest released version.

## Reporting a vulnerability

Please do not open public GitHub issues for suspected security vulnerabilities.

Report security issues privately by contacting:

- GitHub: open a private security advisory if enabled for this repository
- Or email the maintainer directly (add preferred address in this file)

Include:

- Description of the issue
- Reproduction steps
- Affected version(s)
- Potential impact
- Any suggested fix or mitigation

## Response targets

- Initial acknowledgment: within 7 days
- Status update after triage: within 14 days when possible

## Disclosure policy

Please allow time for triage and patching before public disclosure.

## Scope notes

- WhyLowDPS is local-first; Battle.net credentials are intended to remain local.
- Reports that show credential exfiltration risk or unintended remote data exposure are high priority.
