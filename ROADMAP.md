# Roadmap

This roadmap reflects current priorities and may change as user needs evolve.

## Near term

- Improve release trust surface:
  - consistent release notes
  - checksums on every release
  - clearer install/runtime warnings
- Continue frontend maintainability work by decomposing large components and centralizing shared logic.
- Keep data/config flows runtime-driven where possible (avoid stale static snapshots).

## Mid term

- Improve onboarding and first-run UX.
- Strengthen settings discoverability for key workflows.
- Expand reliability and error messaging around startup, auth, and external data refresh paths.

## Ongoing

- Keep dependencies current across frontend and Rust workspaces.
- Resolve known security advisories when upstream constraints allow.
- Preserve local-first behavior and transparent privacy posture.

## Non-goals (current)

- Non-Windows desktop support
- Cloud-hosted user-data backend
